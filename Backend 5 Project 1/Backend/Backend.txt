###
Backend -------
The backend is the part of a website or app that works behind the scenes. It handles things users don't see, such as:
---
Receiving requests from the frontend.
Processing data and business logic.
Connecting to databases.
Authenticating users (login/signup).
Sending responses back to the frontend.
---
Example:
On Instagram, when you log in:

Frontend: Shows the login form.
Backend: Checks your email and password in the database and tells the frontend whether you're authenticated.

What is an API?
API (Application Programming Interface) is a set of rules that allows two software applications to communicate with each other.

In simple words:
An API acts as a messenger between a client and a server. The client sends a request, and the API returns a response.
FlowChart:::

Client (Browser/App)
        │
     Request
        │
       API

    Database
        │
     Response
        │

#####
Type OF APIs---

### **1. Public API (Open API)**
A **Public API** is available for anyone to use. Developers can access it to build applications, usually by signing up for an API key.
**Example:** OpenWeather API, Google Maps API.

---

### **2. Private API**
A **Private API** is used only within an organization. It helps different internal systems and teams communicate securely.
**Example:** A company's employee management or payroll system.

---

### **3. Partner API**
A **Partner API** is shared only with authorized business partners. Access is restricted and usually requires authentication or an agreement.
**Example:** A payment gateway API provided to partner websites.

---

### **4. Composite API**
A **Composite API** combines multiple API requests into a single request. This reduces the number of network calls and improves performance.
**Example:** Fetching a user's profile, orders, and cart with one request.

---

### **5. REST API**
A **REST API** is the most widely used type of web API. It uses HTTP methods like **GET, POST, PUT, and DELETE** and typically exchanges data in JSON format.
**Example:** An Express.js API that returns a list of users.

---

### **6. GraphQL API**
A **GraphQL API** allows clients to request exactly the data they need. This avoids receiving unnecessary data and makes data fetching more efficient.
**Example:** A mobile app requesting only a user's name and profile picture.

---

### **7. SOAP API**
A **SOAP API** is a protocol that uses XML to exchange data. It is highly secure and commonly used in enterprise applications such as banking and healthcare.
**Example:** Online banking or insurance systems.

---

### **8. gRPC API**
**gRPC** is a high-performance API framework developed by Google. It uses Protocol Buffers instead of JSON, making communication faster and more efficient.
**Example:** Communication between microservices in large-scale applications.

HTTP Status Codes (Main Categories)
HTTP status codes are **3-digit numbers** returned by the server to tell the client (browser, Postman, mobile app, etc.) what happened to the request.

## 1xx – Informational

The request has been received, and the server is continuing to process it.

* **100 Continue** – The client can continue sending the request.
* **101 Switching Protocols** – The server is switching to another protocol (e.g., WebSocket).

---

## 2xx – Success

The request was successful.

* **200 OK** – The request was successful, and the server returned the requested data.
* **201 Created** – A new resource was successfully created (commonly used with `POST`).
* **202 Accepted** – The request has been accepted for processing but is not completed yet.
* **204 No Content** – The request succeeded, but there is no data to return.

---

## 3xx – Redirection

The client needs to take another action to complete the request.

* **301 Moved Permanently** – The resource has been permanently moved to a new URL.
* **302 Found** – The resource is temporarily available at another URL.
* **304 Not Modified** – The cached version is still valid, so no new data is sent.
* **307 Temporary Redirect** – Temporary redirect while preserving the original HTTP method.
* **308 Permanent Redirect** – Permanent redirect while preserving the original HTTP method.

---

## 4xx – Client Errors

The request has a problem caused by the client.

* **400 Bad Request** – The request is invalid or contains incorrect data.
* **401 Unauthorized** – Authentication is required or the credentials are invalid.
* **403 Forbidden** – The client is authenticated but does not have permission.
* **404 Not Found** – The requested resource does not exist.
* **405 Method Not Allowed** – The HTTP method (GET, POST, etc.) is not allowed.
* **406 Not Acceptable** – The server cannot provide a response matching the requested format.
* **408 Request Timeout** – The client took too long to send the request.
* **409 Conflict** – The request conflicts with the current state of the resource.
* **410 Gone** – The resource has been permanently removed.
* **413 Payload Too Large** – The request body exceeds the allowed size.
* **415 Unsupported Media Type** – The server does not support the request's content type.
* **422 Unprocessable Content** – The request format is correct, but the data fails validation.
* **429 Too Many Requests** – The client has sent too many requests in a short time (rate limit exceeded).

---

## 5xx – Server Errors

The request was valid, but the server failed to process it.

* **500 Internal Server Error** – A generic server error occurred.
* **501 Not Implemented** – The server does not support the requested functionality.
* **502 Bad Gateway** – A gateway or proxy received an invalid response from another server.
* **503 Service Unavailable** – The server is temporarily unavailable (maintenance or overload).
* **504 Gateway Timeout** – A gateway or proxy timed out waiting for another server.
* **505 HTTP Version Not Supported** – The server does not support the HTTP version used.

# What is an HTTP Server?

An **HTTP server** is software that receives HTTP requests from clients (such as browsers, mobile apps, or Postman), processes them, and sends back an HTTP response.

Examples:

* Express.js (Node.js)
* Apache HTTP Server
* Nginx
* Microsoft IIS

```
   │ Processes request
   ▼

   │
   ▼

   │
   │ 200 OK + JSON data
   ▼

```

# What is an HTTPS Server?

An **HTTPS server** is an HTTP server that uses **SSL/TLS encryption** to secure communication between the client and the server. Data is encrypted so others cannot read or modify it during transmission.

Benefits:

* Encrypts passwords, payment details, and personal information.
* Verifies the website's identity using an SSL/TLS certificate.
* Protects against man-in-the-middle attacks.
* Required for many modern browser features.

---

## HTTP vs HTTPS

| HTTP                       | HTTPS                           |
| -------------------------- | ------------------------------- |
| Data is sent in plain text | Data is encrypted               |
| Less secure                | Highly secure                   |
| Uses port **80**           | Uses port **443**               |
| URL starts with `http://`  | URL starts with `https://`      |
| No SSL/TLS certificate     | Requires an SSL/TLS certificate |

## Example Express HTTP Server

```javascript
const express = require("express");
const app = express();

app.get("/", (req, res) => {
  res.status(200).send("Hello World");
});

app.listen(3000, () => {
  console.log("HTTP Server running on port 3000");
});
```

---

## Example Express HTTPS Server

```javascript
const express = require("express");
const https = require("https");
const fs = require("fs");

const app = express();

app.get("/", (req, res) => {
  res.send("Secure HTTPS Server");
});

const options = {
  key: fs.readFileSync("private.key"),
  cert: fs.readFileSync("certificate.crt")
};

https.createServer(options, app).listen(443, () => {
  console.log("HTTPS Server running on port 443");
});
```

### Database Operations (CRUD) – Short Notes

* **Create** → Add new data to the database.

  * HTTP Method: `POST`
  * Example: Create a new user.

* **Read** → Retrieve data from the database.

  * HTTP Method: `GET`
  * Example: Get all users or a specific user.

* **Update** → Modify existing data.

  * HTTP Methods: `PUT` (replace) or `PATCH` (partial update)
  * Example: Change a user's email.

* **Delete** → Remove data from the database.

  * HTTP Method: `DELETE`
  * Example: Delete a user by ID.

### CRUD Summary

| Operation | HTTP Method     | Purpose      |
| --------- | --------------- | ------------ |
| Create    | `POST`          | Add new data |
| Read      | `GET`           | Fetch data   |
| Update    | `PUT` / `PATCH` | Modify data  |
| Delete    | `DELETE`        | Remove data  |

**Easy way to remember:**

* **POST** → Create
* **GET** → Read
* **PUT/PATCH** → Update
* **DELETE** → Delete

 ## Mongoose (Short Notes)

**Mongoose** is an **ODM (Object Data Modeling)** library for **Node.js** that makes it easier to work with **MongoDB**.

### Key Points

* Connects a Node.js application to MongoDB.
* Defines the structure of data using **Schemas**.
* Creates **Models** to interact with collections.
* Provides built-in **validation** and CRUD methods.
* Makes database operations simpler than using the native MongoDB driver.

### Basic Flow

```text
Node.js/Express
      │
      ▼
   Mongoose
      │
      ▼
   MongoDB
```

### Common Methods

* `connect()` → Connect to MongoDB
* `create()` → Insert data
* `find()` → Get all data
* `findById()` → Get data by ID
* `findByIdAndUpdate()` → Update data
* `findByIdAndDelete()` → Delete data
* `save()` → Save a document

### Example

```javascript
const mongoose = require("mongoose");

mongoose.connect("mongodb://127.0.0.1:27017/myDB");
```
**One-line definition (Interview):**
**Mongoose is an ODM library for Node.js that simplifies working with MongoDB by using schemas,
models, and built-in methods for database operations.**


###
## Schema (Short Definition)

A **Schema** is a **blueprint** that defines the structure of documents in a MongoDB collection. It specifies what fields a document has, their data types, and validation rules.

### Example

```javascript
const userSchema = new mongoose.Schema({
  name: String,
  age: Number,
  email: String
});
```

Here, every user document will have:

* `name` → String
* `age` → Number
* `email` → String

### One-line Definition (Interview)

 **A Schema is a blueprint that defines the structure, data types, and validation rules for documents in a MongoDB collection.**
