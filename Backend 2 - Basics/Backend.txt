###
Backend -------
The backend is the part of a website or app that works behind the scenes. It handles things users don't see, such as:
---
Receiving requests from the frontend.
Processing data and business logic.
Connecting to databases.
Authenticating users (login/signup).
Sending responses back to the frontend.
---
Example:
On Instagram, when you log in:

Frontend: Shows the login form.
Backend: Checks your email and password in the database and tells the frontend whether you're authenticated.


###
Node Package Manager (npm) -----

npm (Node Package Manager) is a tool used to download, install,
update, remove, and manage packages (libraries) that other developers have published.

A package is a collection of reusable code that solves a specific problem, so you don't have to write everything from scratch.


###
Node.js
Node.js is an open-source, cross-platform JavaScript runtime environment that allows you to run JavaScript outside the browser, mainly on the server. It is used to build fast, scalable backend applications, APIs, and web servers.

In simple words:

Node.js lets developers use JavaScript to build the backend of web applications.

Example:
With Node.js, you can:

Create a web server.
Build REST APIs.
Connect to databases.
Handle user authentication.
Build real-time applications like chat apps.


###
Express.js (Short Definition)

Express.js is a fast, lightweight, and flexible web framework for Node.js that simplifies building web applications and APIs. It provides tools for handling routes, requests, responses, middleware, and server-side logic.

In simple words:

Express.js makes it easier and faster to build backend applications with Node.js.

Example:

With Express.js, you can:

Create web servers.
Build REST APIs.
Handle routes (URLs).
Process HTTP requests and responses.
Use middleware for authentication, logging, and more.

API
What is an API?
API (Application Programming Interface) is a set of rules that allows two software applications to communicate with each other.

In simple words:
An API acts as a messenger between a client and a server. The client sends a request, and the API returns a response.
FlowChart:::

Client (Browser/App)
        │
     Request
        │
       API
        │
     Server
        │
    Database
        │
     Response
        │
Client (Browser/App)
 
#####
Type OF APIs---

### **1. Public API (Open API)**
A **Public API** is available for anyone to use. Developers can access it to build applications, usually by signing up for an API key.
**Example:** OpenWeather API, Google Maps API.

---

### **2. Private API**
A **Private API** is used only within an organization. It helps different internal systems and teams communicate securely.
**Example:** A company's employee management or payroll system.

---

### **3. Partner API**
A **Partner API** is shared only with authorized business partners. Access is restricted and usually requires authentication or an agreement.
**Example:** A payment gateway API provided to partner websites.

---

### **4. Composite API**
A **Composite API** combines multiple API requests into a single request. This reduces the number of network calls and improves performance.
**Example:** Fetching a user's profile, orders, and cart with one request.

---

### **5. REST API**
A **REST API** is the most widely used type of web API. It uses HTTP methods like **GET, POST, PUT, and DELETE** and typically exchanges data in JSON format.
**Example:** An Express.js API that returns a list of users.

---

### **6. GraphQL API**
A **GraphQL API** allows clients to request exactly the data they need. This avoids receiving unnecessary data and makes data fetching more efficient.
**Example:** A mobile app requesting only a user's name and profile picture.

---

### **7. SOAP API**
A **SOAP API** is a protocol that uses XML to exchange data. It is highly secure and commonly used in enterprise applications such as banking and healthcare.
**Example:** Online banking or insurance systems.

---

### **8. gRPC API**
**gRPC** is a high-performance API framework developed by Google. It uses Protocol Buffers instead of JSON, making communication faster and more efficient.
**Example:** Communication between microservices in large-scale applications.
